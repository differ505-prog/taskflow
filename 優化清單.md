### 🔧 P1 - 手機版 TaskDetailPanel 子任務按鈕「按兩次才打勾」

**問題描述**（2026-07-16 用戶回報）

- **症狀**：手機版「任務詳情面板」內的子任務圈圈（`TaskDetailPanel.tsx` 第 411 行 `<button>`）需要點兩次才會打勾
- **桌面版**：點一次即可正常 toggle（已驗證）
- **任務清單區**：點一次即可正常 toggle（已驗證）
- **關鍵 user 線索**：若先用手指點「子任務文字」（讓焦點切到該行），再點圈圈，就只需要點一次即可打勾

**懷疑根因（信心約 80%）**

- `TaskDetailPanel.tsx` 第 411-421 行子任務 button 缺 `e.stopPropagation()`
- 手機版 detail panel 是 `fixed inset-0 z-[60]`（`AppLayout.tsx` 第 132、192 行），祖先層的某個 onClick（如 motion 的拖曳偵測、panel 的 swipe handler）可能在第一次 click 時攔截事件並 e.preventDefault()
- 桌面版因 panel 是 `w-[480px]` 的右側欄，祖先層級 click 不會跟子按鈕衝突

**預定修法（候選）**

1. 最便宜：給子任務 `<button onClick>` 的 handler 第一行加 `e.stopPropagation()`，並用 `e.preventDefault()` 防止 mobile Safari 的 double-tap-to-zoom 干擾
2. 若無效：檢查 panel root 是否需要 `e.target === e.currentTarget` 條件判斷
3. 若仍無效：在 mobile 開 chrome://inspect 印 log 確認是否真的「同一個 click event 被觸發兩次 toggle」

**驗收標準**

- 手機版「任務詳情面板」中，第一次點子任務圈圈就直接打勾 ✓
- 任務卡其他行為（點文字進入編輯、刪除子任務）不受影響

## 核心 UX 原則

1. **Immediate Revelation** — 資訊優先可見，減少點擊
   - 子任務預設展開，不需要展開/折疊操作
   - 點擊任務卡片 → 開啟編輯面板

2. **Smart Defaults** — 智慧分類減少手動操作
   - Today / Upcoming / Flagged 等自動過濾視圖

3. **Visual Hierarchy** — 清晰的視覺層級
   - 已完成任務：灰階 + 刪除線
   - 任務間距充足，呼吸感強
