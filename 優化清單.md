## OmniSonic 整合紀錄（跨 repo：differ505-prog/music-focus-environment）

> 整合時間：2026-07-19
> 整合目的：Pomodoro Timer focus 模式下嵌入 OmniSonic Deep Focus 按鈕，點擊後跨 iframe postMessage 觸發 ZenFlowPanel 自動 DJ 播放。
> 環境變數：`NEXT_PUBLIC_OMNISONIC_URL=https://music-focus-environment.vercel.app`（Wade Vercel production 已設定）
> 通訊協議：embed page 發送 `OMNI_START_RANDOM` / `OMNI_PLAYING` / `OMNI_ERROR` postMessage，Wade 端 `ZenFlowContext` 監聽

### Phase A：OmniSonic embed page 移除 GlobalPlayer UI（避免入侵 Wade iframe）

- **commit hash**：`adf8c52`（位於 music-focus-environment repo）
- **問題**：embed page `src/app/layout.tsx` 標 `'use client'` 與 `export const metadata` 衝突 → Vercel build 失敗 → 上一版 `f6a7f72` 的 embedMode 改動實際未部署、SSR HTML 仍含 GlobalPlayer
- **根因**（命中 §26 已知類別 + §25 既有 pattern 對齊）：server-only API（metadata）與 client-side API（usePathname）混用
- **修法**：拆 layout 為 server component（保留 metadata export）+ 新增 `src/components/embed-route-guard.tsx`（client component 內含 usePathname 判斷）
- **驗證**：curl `https://music-focus-environment.vercel.app/embed/button` → SSR HTML 只含「開始深度專注心流」按鈕、GlobalPlayer 完全消失 ✅
- **§26 命中**：客戶端/伺服器元件邊界 + metadata server-only

### Phase B：Wade iframe 整合手動驗收

- **驗收方式**：你親自登入 `taskflow-v2.vercel.app` → 進 Pomodoro Timer（focus 模式）→ 看到 OmniSonic 嵌入按鈕 → 點擊測試 → 確認播放/暫停/切歌 + ZenFlowPanel 展開
- **驗收結果**：✅ 整合正常、跨 iframe postMessage 通訊成功、音訊播放成功
- **未自動化驗證原因**：登入需 Firebase auth credentials，屬 §15.1 blocker；MCP `browser_lock` 被 Auto-review 阻擋；runtime tool call 預算已用 3/6

## 核心 UX 原則

1. **Immediate Revelation** — 資訊優先可見，減少點擊
   - 子任務預設展開，不需要展開/折疊操作
   - 點擊任務卡片 → 開啟編輯面板

2. **Smart Defaults** — 智慧分類減少手動操作
   - Today / Upcoming / Flagged 等自動過濾視圖

3. **Visual Hierarchy** — 清晰的視覺層級
   - 已完成任務：灰階 + 刪除線
   - 任務間距充足，呼吸感強
